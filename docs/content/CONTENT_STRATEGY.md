# Hub Content Strategy

Hub content should make Starknet exploration understandable, safe, and locally relevant.

## Content Goals

Hub content should:

- introduce Starknet resources with enough context;
- reduce confusion for non-technical learners;
- avoid speculative or trading-first framing;
- help learners understand what to open next;
- bridge users toward community and builder paths.

## Content Categories

| Category | Purpose |
|---|---|
| Wallet Safety | Help learners avoid seed phrase, signature, phishing, and backup mistakes |
| Starknet Basics | Explain Starknet, account abstraction, STARKs, fees, and ecosystem language |
| Guided Resources | Curated docs, tutorials, apps, and learning material |
| Missions | Step-by-step exploration tasks |
| Community | Local and global community entry points |
| Builder Path | Cairo, Scarb, Starknet Foundry, Dojo, and developer resources |
| Local Use Cases | Public-goods and community-first applications |

## Writing Style

Use clear, action-oriented language.

Good:

```text
Pelajari dasar Starknet
Ikuti misi pertama
Buka resource ini setelah memahami wallet safety
```

Avoid:

```text
Runtime ecosystem execution
Protocol utility surface
Connect wallet immediately
```

## Every Resource Should Include

- title;
- short description;
- readiness level;
- why it matters;
- when to use it;
- safety note if needed;
- destination link.

## External Links

External links should be curated and purposeful. Avoid dumping many links without explanation.

## Financial Safety

Do not frame Hub as investment advice, token recommendation, trading guide, or profit opportunity.
